### สิ่งที่ปรับใน Carbon Analytics รอบล่าสุด

- เอาหน้า Pipeline Proof ออกจากเมนูและ route ที่ผู้ใช้เข้าถึง
- เอาส่วนกระบวนการขนส่งออกจาก dashboard หลัก และเหลือเฉพาะกระบวนการเพาะปลูก
- ปรับเมนู `ภาพรวม Carbon` ให้เป็น dropdown 3 รายการ:
  - `ข้อมูลสรุป`
  - `กระบวนการเพาะปลูก`
  - `รายงาน Premium T-VER`
- คืนหน้า `ข้อมูลสรุป` กลับมาเป็นหน้าสรุปภาพรวม Carbon โดยไม่แสดงแผนที่ประเทศไทยในหน้านี้
- ปรับ KPI ด้านบนของหน้าสรุปให้เหลือข้อมูลที่ต้องการ:
  - Carbon รวม
  - Machine/Fuel หรือน้ำมันที่ใช้ทั้งหมด
  - จำนวนแปลงทั้งหมดที่ร่วมโครงการ
  - จำนวนการใช้ปุ๋ยทั้งหมด
  - ผลต่างการปล่อย Carbon เทียบค่าเฉลี่ยปีฐาน
- เพิ่มสรุปค่าเฉลี่ยปีฐาน, ปีที่ดำเนินการ และผลการลด/เพิ่ม Carbon
- เพิ่มกราฟเส้น `CO2e รวมรายปี เทียบค่าเฉลี่ยปีฐาน`
- เพิ่มสรุปและกราฟเปรียบเทียบสัดส่วนการใช้น้ำมันและปุ๋ยระหว่างปีฐานกับปีที่ดำเนินการ

### สิ่งที่ปรับในหน้ากระบวนการเพาะปลูก

- เหลือ 4 ขั้นตอนตาม requirement:
  - เตรียมดิน
  - เพาะปลูก
  - ดูแลรักษา
  - เก็บเกี่ยว
- ปรับกราฟวงกลมสัดส่วนกระบวนการให้แสดงเฉพาะ 4 ขั้นตอนนี้
- ปรับกราฟแท่งเปรียบเทียบค่าเฉลี่ยปีฐานกับปีดำเนินการให้เหลือเฉพาะ 4 ขั้นตอน
- เพิ่มกราฟเปรียบเทียบการใช้ปุ๋ยและน้ำมันรายกระบวนการ เพื่อดูว่าสัดส่วน input ในแต่ละขั้นตอนเป็นอย่างไร

### สิ่งที่ปรับในหน้าแผนที่และรายละเอียดรายพื้นที่

- เปลี่ยนตัวเลือกพื้นที่จากปุ่มกดเป็น dropdown
- ค่าเริ่มต้นเป็น `ทั้งหมด`
- รองรับการเลือกตามลำดับพื้นที่ เช่น ภาค, จังหวัด, อำเภอ/เขต, ตำบล/แขวง และแปลง
- หน้าแผนที่ยังแยกอยู่เป็นหน้ารายละเอียดพื้นที่ ไม่ได้แสดงในหน้าสรุปภาพรวม Carbon แล้ว

### สิ่งที่ปรับในรายงาน Premium T-VER

- คงหน้าสรุปบนเว็บไว้เป็นรูปแบบตาราง/กล่องข้อมูลที่อ่านง่ายเหมือนเดิม
- ลดความหนาของตัวอักษรในกล่องข้อมูลบนเว็บให้ดูโปร่งขึ้น
- เพิ่ม Preview และ Download สำหรับ:
  - PDF
  - Word
  - Excel
- ปรับปุ่ม Download PDF เป็นโทนส้มแดงตาม mood ของไอคอน PDF
- ปรับ PDF ให้ใช้รูปแบบสรุปตามตัวอย่างเอกสาร PDD ที่แนบมา โดยอ้างอิงโครงสร้างเนื้อหาจากหัวข้อสรุปปริมาณการลดก๊าซเรือนกระจกและสรุปปริมาณก๊าซเรือนกระจกที่คาดว่าจะลดได้
- PDF ไม่ใส่เลขหัวข้อ `3.5` หรือ `3.6` ตามที่แก้ล่าสุด
- แก้ layout PDF ให้ตารางไม่แตก และแยกส่วน `สรุปปริมาณก๊าซเรือนกระจกที่คาดว่าจะลดได้` ไปขึ้นหน้าใหม่

### สถานะข้อมูล Mock และผลกระทบกับงานเพื่อน

ตอนนี้ dashboard ใช้ข้อมูลจำลองเพื่อให้เห็นหน้าตา dashboard และกราฟก่อนที่ข้อมูล input จริงจะเสร็จ โดยข้อมูลจำลองจำกัดอยู่ในโซน frontend ของ Carbon Analytics เท่านั้น

- ไฟล์ mock หลักอยู่ใน `frontend/src/features/cf-dashboard/data/mockDashboard.ts`
- จุดสลับระหว่าง mock กับ API อยู่ใน `frontend/src/features/cf-dashboard/services/dashboardApi.ts`
- ค่า `ENABLE_API_DASHBOARD = false` ใช้สำหรับ preview หน้าตา dashboard ชั่วคราว
- ไม่มีการเขียนข้อมูล mock ลงฐานข้อมูล
- ไม่แก้ schema ฐานข้อมูล
- ไม่กระทบ backend API ของเพื่อน
- เมื่อข้อมูลจริงพร้อม สามารถกลับไปใช้ API เดิมได้โดยสลับ flag และเชื่อมข้อมูลจริงต่อจาก logic ที่ comment/โครงสร้างเดิมเก็บไว้

### อัปเดตเพิ่มเติมหลังปรับ UI กราฟและหน้าแผนที่

มีการปรับเพิ่มหลังจากรอบ merge `block_dev` เพื่อให้ dashboard อ่านง่ายขึ้นและอธิบายข้อมูลรายพื้นที่ได้ละเอียดขึ้น

สิ่งที่ปรับเพิ่มในหน้าภาพรวม Carbon:

- ปรับรายการสรุปใต้กราฟเปรียบเทียบ 4 ขั้นตอนให้เรียงตามปริมาณการปล่อยของปีดำเนินการจากมากไปน้อย
- เพิ่ม badge ระบุชัดเจนว่ากระบวนการไหน `ปล่อยมากสุด` และ `ปล่อยน้อยสุด`
- ยังคงลำดับกราฟแท่งตามขั้นตอนกระบวนการ `1-4` เพื่อไม่ให้สับสนกับลำดับการทำงานจริง
- เพิ่มข้อความสรุปกระบวนการที่ปล่อยสูงสุดและต่ำสุด พร้อมค่า tCO2e

สิ่งที่ปรับเพิ่มเรื่องสีกราฟ:

- เปลี่ยนสีกราฟแท่งและกราฟเส้นจากเขียว/ส้มเดิมเป็นโทน pastel
- เพิ่ม palette กลางใน `frontend/src/features/cf-dashboard/components/charts/ChartRegistry.ts`
- ให้กราฟแท่ง, กราฟเส้น, กราฟวงกลม และกราฟ input ใช้โทนสีไปในแนวเดียวกัน
- ปรับสี baseline เป็นฟ้าอมม่วง pastel และ project year เป็น coral/pink pastel
- ปรับสีกราฟปุ๋ยและน้ำมันให้แยกกันชัดเจนขึ้นด้วยโทนม่วงชมพูและฟ้าอมเขียว

สิ่งที่ปรับเพิ่มในหน้าแผนที่และรายละเอียดรายพื้นที่:

- เพิ่มสรุปการใช้ปุ๋ยและน้ำมันของพื้นที่ที่เลือก
- แสดงค่าเปรียบเทียบปีฐานและปีดำเนินการ ได้แก่:
  - ปุ๋ยปีฐาน
  - ปุ๋ยปีดำเนินการ
  - น้ำมันปีฐาน
  - น้ำมันปีดำเนินการ
- เพิ่มผลต่างว่าปุ๋ยและน้ำมันลดลงหรือเพิ่มขึ้นเท่าไหร่ พร้อมเปอร์เซ็นต์
- เพิ่มกราฟแท่งรวม `ปุ๋ยและน้ำมันรวม · ปีฐาน vs ปีดำเนินการ`
- เพิ่มตารางเจาะรายกระบวนการ 4 ขั้นตอน เพื่อดูว่าขั้นตอนใดใช้ปุ๋ย/น้ำมันเท่าไหร่ในปีฐานและปีดำเนินการ
- เพิ่มกราฟแท่งรายกระบวนการสำหรับปุ๋ยและน้ำมัน เพื่อใช้ประกอบการอธิบายเชิงพื้นที่

เหตุผลในการเลือกกราฟ:

- ใช้ grouped bar สำหรับการเทียบปีฐานกับปีดำเนินการ เพราะเห็นความต่างของปริมาณได้ชัดกว่า pie chart
- ใช้ตารางสำหรับข้อมูลเจาะรายกระบวนการ เพราะมีหลายมิติ เช่น ปุ๋ยปีฐาน, ปุ๋ยปีดำเนินการ, ผลต่าง, น้ำมันปีฐาน, น้ำมันปีดำเนินการ และผลต่าง
- ยังใช้ doughnut chart สำหรับสัดส่วน Carbon รายกระบวนการ เพราะเหมาะกับการดูสัดส่วนภาพรวมของพื้นที่

สถานะข้อมูล mock รายพื้นที่:

- เมื่อ API จริงพร้อม สามารถส่งข้อมูลรายพื้นที่ใน field เดียวกันเพื่อแทน mock ได้

## ภาพรวมงานที่ทำ

งานส่วนนี้เป็นการพัฒนาหน้า Dashboard สำหรับสรุปผล Carbon Footprint โดยเน้นการนำข้อมูลจริงจากฐานข้อมูล PostgreSQL มาแสดงผล ไม่ใช้ข้อมูล Mock เป็นข้อมูลหลักอีกต่อไป ระบบจะดึงข้อมูลผ่าน NestJS API ในโมดูล `analytics` แล้วนำมาแสดงในหน้าเว็บฝั่ง React

นิยามข้อมูลปีที่ใช้ใน Dashboard:

- ปีล่าสุดที่พบในฐานข้อมูล = ปีดำเนินโครงการ
- ปีก่อนหน้าทั้งหมด = ปีฐาน
- ค่าเฉลี่ยของปีก่อนหน้าทั้งหมด = ค่าเฉลี่ยปีฐาน

## หน้าที่เพิ่มและปรับปรุง

### 1. หน้า `/overview` ภาพรวม Carbon

ปรับหน้า Overview ให้แสดงข้อมูลจริงจากฐานข้อมูล และจัด layout ใหม่ให้เหมาะกับการสรุปผลภาพรวมมากขึ้น

สิ่งที่ทำ:

- แสดง KPI จากข้อมูลจริง เช่น Carbon รวม, Machine/Fuel, Input CO2e, จำนวนเกษตรกร และจำนวนแปลง
- เพิ่มกราฟ Trend แบบเต็มความกว้างของหน้า เพื่อเปรียบเทียบปีฐานรายปีกับปีดำเนินโครงการ
- เพิ่มเส้นค่าเฉลี่ยปีฐานในกราฟ Trend
- จัด Block สัดส่วน 5 ขั้นตอน และ Grouped Bar ให้อยู่ข้างกัน
- ปรับแผนภูมิวงกลมให้มีตัวเลขแสดงถาวร ไม่ต้องรอ hover
- เพิ่มแผนที่ประเทศไทยขนาดใหญ่ด้านล่าง
- ใช้สีแสดงผลเพิ่ม/ลด Carbon ตามที่กำหนด:
  - ลดลง: `#277B27`
  - เพิ่มขึ้น: `#BA0900`

### 2. หน้า `/process` กระบวนการเพาะปลูก

แยกส่วนกระบวนการเพาะปลูกออกจากการขนส่ง เพื่อให้ดูข้อมูลเฉพาะการเพาะปลูกได้ชัดเจนขึ้น

สิ่งที่ทำ:

- แสดงเฉพาะข้อมูลกระบวนการเพาะปลูก
- เลือกดูได้ทั้งค่าเฉลี่ยปีฐาน รายปีฐาน และปีดำเนินโครงการ
- เพิ่มแผนภูมิวงกลมกระบวนการทั้งหมด พร้อมตัวเลขประกอบ
- เพิ่ม Grouped Bar เปรียบเทียบค่าเฉลี่ยปีฐานกับปีดำเนินโครงการ
- เพิ่มข้อความสรุปว่ากระบวนการใดเพิ่มขึ้นหรือลดลงกี่ tCO2e
- เพิ่มแผนภูมิวงกลมแยกตามแต่ละกระบวนการ พร้อมตัวเลขประกอบ

### 3. หน้า `/transport` กระบวนการขนส่งเข้าโรงงาน

เพิ่มหน้าใหม่สำหรับแสดงข้อมูลการขนส่งเข้าโรงงานโดยเฉพาะ

สิ่งที่ทำ:

- แยกข้อมูลขนส่งออกจากกระบวนการเพาะปลูก
- มีตัวเลือกปีเหมือนหน้า Process
- แสดง Pie Chart และ Bar Chart จากข้อมูลจริง
- แสดงตัวเลขบนแผนภูมิ และข้อความสรุปการเพิ่ม/ลดของค่าการปล่อย Carbon

### 4. หน้า `/spatial` แผนที่ประเทศไทยและรายละเอียดพื้นที่

ปรับหน้าแผนที่ให้ใช้เป็นศูนย์กลางของการ drill-down พื้นที่ และย้ายรายละเอียดสรุปลงด้านล่างตามที่ต้องการ

สิ่งที่ทำ:

- แสดงแผนที่ประเทศไทยเป็น Block ใหญ่ตรงกลาง
- เพิ่มตัวเลือกพื้นที่ด้านบน
- รองรับการ drill-down ตามข้อมูลจริง:
  - ประเทศ
  - ภาค
  - จังหวัด
  - อำเภอ
  - ตำบล
  - รายแปลง
- แสดงจำนวนแปลง จำนวนเกษตรกร จำนวนไร่ และค่า Carbon ของพื้นที่ที่เลือก
- เพิ่ม Pie Chart สัดส่วนกระบวนการของพื้นที่
- เพิ่ม Bar Chart เปรียบเทียบค่าเฉลี่ยปีฐานกับปีดำเนินโครงการของพื้นที่
- ปรับสีหมุดและลูกศรบนแผนที่:
  - ลดลง: `#277B27`

## Backend API ที่เพิ่ม

เพิ่ม API ภายใต้ `/api/analytics` เพื่อให้ Dashboard ใช้ข้อมูลจริงจากฐานข้อมูล

รายการ API:

- `GET /api/analytics/cf-kpi`
- `GET /api/analytics/cf-trend`
- `GET /api/analytics/cf-process`
- `GET /api/analytics/cf-transport`
- `GET /api/analytics/cf-process-activities`
- `GET /api/analytics/cf-spatial-nodes`
- `GET /api/analytics/cf-report-summary`

API เหล่านี้รองรับ filter สำหรับรายงานและพื้นที่:

- `level=all|region|province|district|subdistrict|field`
- `id=<รหัสพื้นที่>`

## การเชื่อมต่อข้อมูลจริง

ข้อมูล Dashboard ใช้ข้อมูลจริงจากตารางหลักในฐานข้อมูล เช่น:

- `log_activities_detail`
- `activities_header`
- `activities_header_type`
- `activities_header_detail_type`
- `lands`
- `farmers`
- `geographies`
- `provinces`
- `districts`
- `subdistricts`

เงื่อนไขสำคัญ:

- ใช้เฉพาะรายการที่มี `log_act_detail_calStatus_id = 2`
- ค่า Carbon ใช้จาก `log_act_detail_volumeAll`
- ปีใช้จาก `activities_header_startDate`

## คำสั่งที่ใช้ทดสอบ

คำสั่งที่รันเพื่อตรวจสอบแล้ว:

```bash
npm run prisma:generate --workspace=backend
npm run build --workspace=backend
npm run build --workspace=frontend
npm run build
```

ผลลัพธ์:

- Backend build ผ่าน
- Frontend build ผ่าน
- Build รวมทั้งโปรเจกต์ผ่าน

หมายเหตุ:

- มี Vite warning เรื่อง bundle ใหญ่ เพราะเพิ่ม library สำหรับ PDF และ Excel
- มี npm audit warning จาก dependencies เดิม/ใหม่ แต่ยังไม่ได้แก้ด้วย `npm audit fix` เพราะอาจกระทบ dependency graph

## ขอบเขตงานของส่วนนี้

งานนี้เน้นเฉพาะส่วนแสดงผลและสรุปผล Carbon Dashboard ไม่ได้ทำส่วนบันทึกหรือนำเข้าข้อมูลกิจกรรมโดยตรง ส่วนการนำเข้าข้อมูลจริงและการจัดการข้อมูลต้นทางยังเป็นงานอีกส่วนหนึ่งของระบบ แต่ Dashboard สามารถนำข้อมูลที่มีอยู่แล้วในฐานข้อมูลมาแสดงผล วิเคราะห์ และสรุปรายงานได้
